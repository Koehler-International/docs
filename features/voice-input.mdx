## Overview

MobileCoder supports **voice input** for prompts, allowing you to speak naturally to describe what you want to build. This is perfect for complex descriptions, rapid iteration, or hands-free development sessions.

## How to Use Voice Input

<Steps>
<Step title="Click the Microphone Icon">

In the chat input area, click the **microphone icon** to start recording.

</Step>

<Step title="Speak Your Prompt">

Describe what you want to build in natural language. Speak clearly and at a normal pace.

</Step>

<Step title="Stop Recording">

Click the microphone icon again to stop recording. Your speech will be transcribed automatically.

</Step>

<Step title="Review and Send">

Review the transcribed text, make any edits if needed, and send your prompt.

</Step>
</Steps>

## Supported Audio Formats

Voice input supports the following formats:

- MP3
- MP4
- M4A
- WAV
- WebM
- OGG

<Info>
Audio files are limited to 25MB. For longer recordings, consider breaking them into smaller segments.
</Info>

## Best Practices for Voice Input

<AccordionGroup>
  <Accordion icon="microphone" title="Speak clearly and naturally">
    Use a normal speaking pace. The AI transcription works best with clear, natural speech.
  </Accordion>
  <Accordion icon="list" title="Structure your thoughts">
    Before recording, think about what you want to say. A structured description leads to better results.
  </Accordion>
  <Accordion icon="volume-high" title="Minimize background noise">
    Record in a quiet environment for the most accurate transcription.
  </Accordion>
  <Accordion icon="edit" title="Review before sending">
    Always review the transcription before sending. You can edit any mistakes or add clarifications.
  </Accordion>
</AccordionGroup>

## When to Use Voice vs Text

**Use voice input when:**

- Describing complex UX flows conversationally
- You think faster than you type
- Doing hands-free development
- Brainstorming ideas quickly

**Use text input when:**

- You need precise technical terms
- Copying code snippets or error messages
- Making small, specific edits
- Working in a noisy environment
